# Contributing to TaskFlow Pro

Thank you for your interest in contributing to TaskFlow Pro! This document provides guidelines and instructions for contributing.

## Development Setup

1. Fork the repository
2. Clone your fork: `git clone <your-fork-url>`
3. Create a branch: `git checkout -b feature/your-feature-name`
4. Follow the setup instructions in [docs/SETUP.md](docs/SETUP.md)

## Code Style

### Backend
- Follow ESLint configuration (Airbnb base)
- Use Prettier for formatting
- Write meaningful commit messages
- Add JSDoc comments for complex functions

### Frontend
- Follow React best practices
- Use functional components with hooks
- Follow ESLint and Prettier configurations
- Use Tailwind CSS for styling

## Commit Guidelines

Follow conventional commits:
- `feat:` New feature
- `fix:` Bug fix
- `docs:` Documentation changes
- `style:` Code style changes (formatting, etc.)
- `refactor:` Code refactoring
- `test:` Adding tests
- `chore:` Maintenance tasks

Example: `feat: Add drag-and-drop to Kanban board`

## Pull Request Process

1. Ensure your code follows the style guidelines
2. Write or update tests as needed
3. Update documentation if needed
4. Ensure all tests pass
5. Create a descriptive PR with:
   - What changes were made
   - Why the changes were necessary
   - How to test the changes

## Testing

- Write unit tests for new features
- Write integration tests for API endpoints
- Ensure all existing tests pass

## Questions?

Open an issue with the `question` label for any questions or clarifications.
